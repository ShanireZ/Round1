# Question Count Audit Standard

Status: current standard

This document defines the maintained Round1 question-count audit method. Future
question quantity and quality statistics should be generated into `count/` and
linked from plans or standards instead of introducing another counting surface.

For non-question generated inventories, use `count/other-inventories/**`. For
legacy statistic surfaces found elsewhere in the repository, maintain the
migration and retention record in `count/legacy-statistics-index.*`.

## Scope

- Include non-real-paper question bundles under `papers/2026`.
- Exclude official real-paper content from non-real-paper counts.
- Use exam-tagged rows as the unit. If one item supports multiple exam types,
  count one row per exam type.
- Preserve historical reports under `artifacts/reports/**` as evidence only;
  do not treat them as the current counting source.

## Source Inputs

1. `count/state/question-inventory.json`
   - Generated by `npm run inventory:questions -- --write`.
   - Provides raw `examType + questionType + difficulty + kpGroup` available
     counts and blueprint deficits.
2. `count/audits/<runId>/papers-2026__diversity-audit.json`
   - Generated by `audit-question-diversity-2026`.
   - Provides item totals, low-quality candidates, rewrite candidates, template
     clusters, and validation errors.
3. `count/audits/<runId>/papers-2026__rewrite-queue.csv`
   - Candidate items needing rewrite, repair, or regeneration.
4. `count/audits/<runId>/papers-2026__archive-suggestions.csv`
   - Candidate items that are both low quality and in parameterized template
     clusters. These require replacement or manual review before reuse.

## Commands

```powershell
npm run inventory:questions -- --write
npx tsx scripts/audit.ts audit-question-diversity-2026 --dir papers/2026 --out-dir count/audits/<runId>
npm run count:questions -- --write --diversity-audit count/audits/<runId>/papers-2026__diversity-audit.json --rewrite-queue count/audits/<runId>/papers-2026__rewrite-queue.csv --archive-suggestions count/audits/<runId>/papers-2026__archive-suggestions.csv --snapshot-id <snapshotId>
```

If running `tsx` or `npm` inside the local sandbox fails with `spawn EPERM`,
rerun the same command with the required execution permission rather than
changing the audit method.

## Decision Rules

| Situation | Rule |
| --- | --- |
| Current total | Raw non-real-paper available count from inventory |
| Meets current count gate | Bucket has no raw or quality-adjusted deficit |
| Compliant estimate | `available - rewrite`, capped at zero |
| Should abandon | Present in archive suggestions |
| Salvageable | Present in rewrite queue but not archive suggestions |
| Low quality | Rewrite reason includes `qualityScore_below_0.65` |
| Raw deficit | `required - available`, capped at zero per bucket |
| Quality-adjusted deficit | `required - compliant`, capped at zero per bucket |

`lowQuality` overlaps with `salvage` and `abandon`; it is a diagnostic slice,
not an additive status.

## Quality Gates

A candidate is counted as needing rewrite when deterministic diversity audit
flags at least one of:

- `qualityScore_below_0.65`
- hard difficulty rubric failure
- parameterized template cluster
- DS stack/queue overuse candidate

A candidate is counted as an archive suggestion when it is both low quality and
part of a parameterized template cluster. Archive suggestions are not automatic
deletions; they mark items that should be replaced or manually reviewed before
future use.

## Reporting Requirements

Every maintained count report should include:

- per-exam summary
- per-exam and per-question-type summary
- full bucket details for `examType + questionType + difficulty + kpGroup`
- raw and quality-adjusted deficits
- source paths and generated timestamps
- diversity validation error count

The canonical generated files are:

- `count/question-counts-current.md`
- `count/question-counts-current.json`
- `count/snapshots/<snapshotId>.md`
- `count/snapshots/<snapshotId>.json`
- `count/snapshots/<snapshotId>__bucket-details.csv`

Other maintained generated inventories:

- `count/other-inventories/docs/docs-inventory.*`
- `count/other-inventories/papers/papers-inventory.*`
- `count/other-inventories/papers/sections/*`

## Manual Review Boundary

The deterministic audit finds count gaps, template clusters, hard-rubric
failures, and shallow quality signals. Manual or LLM-assisted review is still
required before publishing repaired items, especially for answer ambiguity,
knowledge-point correctness, C++ edge behavior, and explanation quality.
